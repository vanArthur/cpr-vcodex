# Changelog

This changelog starts at `1.2.0.24`, the point where CPR-vCodex began tracking release changes in this file.

| Version | Changes |
|---|---|
| `Unreleased` | - Fixed a race between `Go to %` and EPUB render/status-bar metadata reads that could crash the reader.<br>- Added a streaming fallback for valid EPUB images that the low-memory prefix probe cannot identify, and stopped exposing local PC file paths as broken-image text ([#143](https://github.com/franssjz/cpr-vcodex/issues/143)). |
| `1.3.0.32` | - Improved OTA update reliability when a populated `/.crosspoint` folder leaves too little contiguous heap for HTTPS by running the shared network-memory trim path before update checks and before firmware install/download ([#128](https://github.com/franssjz/cpr-vcodex/issues/128), [#135](https://github.com/franssjz/cpr-vcodex/pull/135)). |
| `1.3.0.31` | - Fixed front-button hint labels so `Up`/`Down` no longer appear swapped in reader menus when `Orient Front Buttons` is enabled with `Landscape CCW` or `Inverted` orientation ([#122](https://github.com/franssjz/cpr-vcodex/issues/122)).<br>- Reduced KOReader Sync/Auth reader-hand-off memory pressure by carrying the already-computed local progress/chapter into sync and formatting credential MD5 bytes without the heap-heavy `String` path ([#124](https://github.com/franssjz/cpr-vcodex/issues/124)).<br>- Reworked flashcard review for large CSV decks so the deck load indexes card keys only, keeps front/back text on the SD card until the active card is displayed, and reports oversized decks/fields cleanly instead of crashing under heap pressure ([#134](https://github.com/franssjz/cpr-vcodex/issues/134)). |
| `1.3.0.30` | - Reduced grayscale/anti-aliased reader RAM pressure by rendering EPUB/TXT pages in row-band strips instead of keeping a full-screen BW backup buffer, with heap telemetry and fallback to the previous path when strip rendering is unavailable.<br>- Made EPUB image and XTC 2-bit grayscale paths use the same tiled grayscale pipeline, skipping off-strip EPUB images during band rendering so image-heavy pages avoid repeated decode/pixel work.<br>- Updated the bundled `open-x4-sdk` to [`198ad26`](https://github.com/crosspoint-reader/community-sdk/commit/198ad267219c25c8ab84418b806c66f1fb5216a3) and applied grayscale base/preconditioning refreshes to XTC page turns and sleep-screen rendering, reducing first-page and overlay ghosting risk on X3/X4. |
| `1.3.0.29` | - Reduced EPUB indexing stalls/crashes on optimized books that repeat presentational images hundreds of times by skipping `role="presentation"`/`aria-hidden` inline images, dropping loaded CSS rules as a low-memory fallback, continuing below the conservative soft heap watermark when hard heap headroom remains, reusing only the last image-dimension probe, suppressing excessive duplicate inline images per chapter, periodically yielding during long section builds, hardening section page deserialization, guarding page turns after failed section loads, and replacing a stuck `Indexing` popup with a minimal page-load error screen if section cache generation still fails. |
| `1.3.0.28` | - Improved KOReader Sync/Auth low-memory resilience by using a shared network-memory trim path that stops NTP, clears font caches, unloads SD-card fonts and their registry metadata, releases reading-stats memory during HTTPS, and restores reader state before returning to normal UI paths ([#124](https://github.com/franssjz/cpr-vcodex/issues/124)).<br>- Fixed EPUB false-completion cases where page navigation in some books could jump to the end-book screen, save 100% progress, and unlock completion achievements before the book was actually finished ([#131](https://github.com/franssjz/cpr-vcodex/issues/131)).<br>- Reduced reader memory pressure with faster filled-rectangle rendering, lower-allocation path normalization, atomic progress-file writes for EPUB/TXT/XTC, and safer progress recovery when files are corrupt or partially written.<br>- Fixed a flashcard deck load memory spike by streaming/parsing large decks with lower transient heap use ([#134](https://github.com/franssjz/cpr-vcodex/issues/134)). |
| `1.3.0.27` | - Improved KOReader Sync reliability on X3/X4 by freeing font and SD-font runtime caches before HTTPS requests, stopping NTP before TLS, retrying transient authentication network errors, and showing more useful low-memory/network details.<br>- Reduced XTC reader memory pressure by reading page-table entries on demand, lazy-loading chapter metadata, closing the XTC file between reads, and streaming 1-bit XTC pages without allocating a full page bitmap buffer.<br>- Guarded ZIP/EPUB inflate reads against no-progress stalls so problematic compressed streams fail cleanly instead of hanging indefinitely during reads or prefix extraction. |
| `1.3.0.26` | - Added Reading Stats persistence protection with an internal `reading_stats.json.bak` restore path and a recovery-mode save guard, reducing the risk of losing heatmap/tracking data after a safe/recovery boot or a broken/empty stats file.<br>- Added configurable Reading Stats autobackups under `Settings > Apps > Reading Stats`, exporting `/exports/stats_backup_YYYY-MM-DD` from the Sync Day date on boot, after Sync Day/manual date changes, and when entering Reading Stats if a first or due backup is missing.<br>- Added confirmed `Clean backups` maintenance for Reading Stats exports, real-file backup detection, popup feedback, and synchronized backup/autobackup translations across all bundled UI languages. |
| `1.3.0.25` | - Improved heavy EPUB page changes by streaming table content as text and deferring image extraction/decoding during indexing, so large books are less likely to get stuck on `Indexing` before the next page renders.<br>- Added lazy ZIP prefix reads and lazy EPUB image extraction so image-heavy chapters keep lower RAM and SD-card pressure, with safer cache rebuilds after the section cache format bump.<br>- Added `Toggle Status Bar` to `Settings > Controls > Short Power Button Click` for EPUB/TXT readers, keeping the reserved status-bar layout space while hiding the bar so pagination stays stable. |
